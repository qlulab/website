
## Lab website image rendering fix
- User previews site via file:// and images didn't show. Diagnosed it was NOT a protocol issue.
- Real bug: directory typo `imgs/gallary/` (extra "a") vs HTML's `imgs/gallery/`; plus lab images were under `campus/` instead of `lab/`.
- Fix: renamed `imgs/gallary` -> `imgs/gallery`, moved lab1/lab2.jpg into `imgs/gallery/lab/`. All gallery refs now resolve.
- Started local static server: `python3 -m http.server 8000 --bind 127.0.0.1` (bg task B1sBM5). Verified gallery.html + images return HTTP 200.
- Lesson: use a local HTTP server (not file://) to faithfully preview before git push; it mirrors GitHub Pages and surfaces path typos.
